MolDynStudio

GUI nativa Windows (PyQt5) + GROMACS rodando em WSL2 via bridge. Arquitetura, fluxo de execução e componentes.

Visão geral Arquitetura Fluxo do usuário WSL Bridge Pipeline GROMACS Módulos Build & Deploy Stack

1. Visão geral

MolDynStudio simula dinâmica molecular de complexos proteína-ligante. A GUI roda nativa no Windows; o motor (GROMACS, AmberTools, MMPBSA, MDAnalysis, MDTraj) roda dentro do WSL2. Tudo é distribuído como MolDynStudio.exe autocontido (PyInstaller --onefile).

Front-end

Windows nativo

PyQt5 + matplotlib. Tabs: Project / MD Setup / MD Run / Analysis. Zero dependência conda no host.

Bridge

WSL2

core/wsl_bridge.py embrulha cada comando como wsl.exe -- bash -lc "conda run -n moldynstudio <cmd>".

Back-end

Linux/Ubuntu

Conda env moldynstudio com GROMACS, AmberTools, gmx_MMPBSA, MDAnalysis, MDTraj, ProDy.

3. Fluxo do usuário (primeiro uso → execução)

1
Duplo-clique em MolDynStudio.exe
PyInstaller extrai environment.yml + install_wsl.ps1 em sys._MEIPASS. main.py sobe.
2
SetupWizard abre como dashboard
3 linhas de status: WSL2 · Conda env · GROMACS. Cada linha tem ícone (✅/❌/⏳) + botão de ação inline.
3
Diagnóstico de virtualização BIOS
_check_virtualization() chama Get-ComputerInfo. Se HyperVRequirementVirtualizationFirmwareEnabled = False, mostra botão BIOS guide em vez de tentar instalar WSL (que falharia).
4
Install WSL2 (se necessário)
Roda install_wsl.ps1 elevado via Start-Process … -Verb RunAs. Habilita VirtualMachinePlatform + Microsoft-Windows-Subsystem-Linux, instala wsl --install --no-distribution, depois Ubuntu-22.04.
5
Create env (se necessário)
_CommandWorker (QThread) roda conda env create -f environment.yml dentro do WSL. Stream stdout pro QTextEdit. ~10-20 min, ~3 GB.
6
Tudo verde → Begin habilitado
Usuário clica Begin →. Dialog fecha. gromacs_analysis_studio_v11.main() abre a MainWindow.
7
Próximas execuções
Launcher abre, todos os checks verdes em <1s. Usuário clica Begin. Comportamento idêntico ao "splash check + launch".

4. Como a WSL Bridge funciona

Wrap de comando

# Chamada Python no Windows:
wsl_bridge.gmx(["pdb2gmx", "-f", "in.pdb"])

# Bridge transforma em:
wsl.exe -- bash -lc \
  "cd /mnt/c/proj && \
   conda run --no-capture-output \
     -n moldynstudio gmx pdb2gmx -f in.pdb"

Conversão de paths

win_to_wsl(r"C:\Users\foo\bar")
# → "/mnt/c/Users/foo/bar"

wsl_to_win("/mnt/c/work/traj.xtc")
# → "C:/work/traj.xtc"

5. Pipeline GROMACS (SystemPrepWorker)

QThread em core/system_prep.py executa 5 etapas sequenciais para preparar o sistema solvatado e neutralizado:

pdb2gmxTopologia + força de campo
→
editconfCaixa de simulação
→
solvateAdiciona moléculas de água
→
gromppPré-processa ions.mdp
→
genionNeutraliza com Na⁺/Cl⁻

Detalhe técnico

  • genion interativo: requer stdin (qual grupo de solvente substituir). Bridge embrulha com echo SOL | gmx genion …
  • ions.mdp auto-gerado: SystemPrepWorker._ensure_ions_mdp() escreve MDP mínimo se ausente
  • Sinais Qt: log = pyqtSignal(str) · done = pyqtSignal(bool, str)

Parametrização de ligante paralela

utils/topology_builder.py · LigandParamWorker (QThread) roda em paralelo ao SystemPrep:

acpype -i ligand.mol2 -c am1bcc -n 0 -b LIG

Gera LIG_GMX.itp + LIG_GMX.gro compatível com AMBER force field.

7. Build & Deploy

Build (1×, desenvolvedor)

Duplo-clique em: build.bat
  • Instala PyInstaller + PyQt5 + matplotlib
  • Roda python build/create_installer.py
  • Produz dist/MolDynStudio.exe (~64 MB)

Run (∞×, usuário final)

Duplo-clique em: MolDynStudio.exe
  • Sem Python instalado no host
  • Sem conda no host
  • WSL2 é instalado pelo próprio app
  • Env conda criado no WSL pelo próprio app

Flags PyInstaller importantes

--onefile                    # single-exe distribuivel
--windowed                   # sem console janela
--add-data environment.yml;. # bundle no _MEIPASS
--add-data install_wsl.ps1;.
--collect-submodules tabs    # captura imports dinamicos
--collect-submodules analysis
--collect-submodules windows
--collect-submodules core
--collect-submodules utils
